# 432 Hz 音频生成与下载平台 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 项目背景
432 Hz 是一种替代标准音乐调音（440 Hz）的频率标准。支持者认为 432 Hz 与宇宙的自然频率更加和谐，能够带来更平和、更舒适的听觉体验。随着冥想、正念和替代疗法的兴起，对 432 Hz 音频内容的需求正在增长。

### 1.2 产品定位
**432-hz.org** 是一个免费的在线平台，允许用户：
- 生成纯净的 432 Hz 音频文件
- 在线试听和预览
- 下载多种格式的音频文件（MP3, WAV, FLAC）
- 无需注册，保护隐私
- 通过广告支持免费服务

### 1.3 目标市场
- **主要市场**：北美和欧洲
- **目标用户群体**：
  - 冥想练习者
  - 瑜伽爱好者
  - 声音疗愈从业者
  - 音乐制作人和音频工程师
  - 对替代调音系统感兴趣的音乐爱好者
  - 寻求放松和减压的普通用户

### 1.4 商业模式
- 广告支持（Google AdSense）
- 非侵入式广告位置
- 未来可扩展：高级功能订阅、API 访问

## 2. 什么是 432 Hz？

### 2.1 科学背景
- **标准调音**：现代音乐标准调音频率为 A4 = 440 Hz（ISO 16 标准，1953年确立）
- **432 Hz 调音**：将 A4 调至 432 Hz，整个音阶相应下调约 8 音分（0.8 个半音）
- **历史依据**：
  - 部分历史学家声称古代文明（埃及、希腊）使用 432 Hz
  - Giuseppe Verdi 曾倡导使用 432 Hz（Verdi's A）
  - 缺乏充分的历史证据支持 432 Hz 是"原始"或"自然"频率

### 2.2 支持者的主张
- 与自然界频率共振（舒曼共振约 7.83 Hz，432 = 7.83 × 55）
- 更和谐、更平和的声音体验
- 有助于冥想、放松和精神集中
- 与人体能量中心（脉轮）对齐

### 2.3 科学局限性
⚠️ **重要声明**：
- 目前缺乏同行评审的科学研究证明 432 Hz 具有特殊疗愈效果
- 大部分声称基于个人体验和主观感受
- 音乐调音标准主要是文化约定，非客观的"正确"或"错误"
- **本平台不提供医疗建议，432 Hz 音频不应替代专业医疗治疗**

## 5. 功能需求

### 5.1 核心功能（MVP）

#### 5.1.1 音频生成器
- **输入参数**：
  - 频率：默认 432 Hz，可自定义（20 Hz - 20 kHz）
  - 时长：5秒 - 60分钟（可输入或预设选项）
  - 波形类型：正弦波（默认）、方波、三角波、锯齿波
  - 音量：0-100%
- **生成方式**：客户端 Web Audio API
- **实时预览**：生成前可试听 3-5 秒片段

#### 5.1.2 音频播放器
- 在线播放控制（播放/暂停）
- 进度条显示和拖动
- 音量控制
- 实时波形可视化（Canvas API 或 Web Audio Analyser）

#### 5.1.3 下载功能
- **支持格式**：
  - MP3（128kbps, 320kbps）
  - WAV（16-bit, 24-bit, 48kHz）
  - FLAC（无损压缩）
- 文件命名：`432hz_[波形类型]_[时长]_[日期].[格式]`
- 下载前预览文件大小

#### 5.1.4 教育内容页面
- "什么是 432 Hz？"科普页面
- 科学观点和争议说明
- 使用建议和最佳实践
- 常见问题解答（FAQ）

### 5.2 辅助功能（V1.1+）

#### 5.2.1 双音拍（Binaural Beats）生成器
- 选择基础频率（如 432 Hz）
- 选择拍频（Delta: 0.5-4 Hz, Theta: 4-8 Hz, Alpha: 8-13 Hz, Beta: 13-30 Hz）
- 生成左右声道不同频率的音频
- 注意：需耳机聆听说明

#### 5.2.2 音频混合器
- 混合 432 Hz 音调与环境音（雨声、海浪、森林）
- 调节各音轨音量比例
- 预设常见组合

#### 5.2.3 音频效果
- 淡入/淡出（防止突兀开始和结束）
- 混响（小房间、大厅、教堂）
- 低通/高通滤波器

#### 5.2.4 历史记录和收藏
- 本地存储（LocalStorage）记录最近生成的配置
- 收藏常用预设
- 一键重新生成

### 5.3 用户体验功能

#### 5.3.1 响应式设计
- 桌面端：优化大屏幕布局，侧边栏参数控制
- 平板端：适配中等屏幕
- 移动端：垂直布局，触摸优化控件

#### 5.3.2 主题切换
- 亮色模式（默认）
- 暗色模式（自动检测系统偏好）
- 手动切换按钮

#### 5.3.3 多语言支持（V2.0+）
- 英语（主要语言）
- 西班牙语
- 德语
- 法语
- 中文

#### 5.3.4 无障碍访问
- WCAG 2.1 AA 级别合规
- 键盘导航支持
- 屏幕阅读器兼容
- 充足的颜色对比度

### 5.4 隐私和安全

- **无用户注册**：完全匿名使用
- **无数据收集**：不存储个人信息

## 6. 技术架构

### 6.2 音频处理

#### 6.2.1 Web Audio API
```typescript
// 核心音频生成逻辑
const audioContext = new AudioContext();
const oscillator = audioContext.createOscillator();
oscillator.frequency.value = 432; // Hz
oscillator.type = 'sine'; // 或 'square', 'triangle', 'sawtooth'
oscillator.connect(audioContext.destination);
oscillator.start();
```

#### 6.2.2 推荐库
- **Tone.js**：高级音频合成和调度
- **Recorder.js** 或 **MediaRecorder API**：录制和导出
- **lamejs**：客户端 MP3 编码
- **audiobuffer-to-wav**：WAV 格式导出

### 11.2 术语表
- **432 Hz**：音乐调音标准，A4 音符调至 432 赫兹
- **双音拍（Binaural Beats）**：两耳接收略有不同频率的音调，大脑感知到拍频
- **波形（Waveform）**：音频信号的形状（正弦、方波、三角、锯齿）
- **采样率（Sample Rate）**：每秒钟采样的次数，常见为 44.1 kHz 或 48 kHz
- **比特深度（Bit Depth）**：每个采样的位数，常见为 16-bit 或 24-bit
